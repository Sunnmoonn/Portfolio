# 1. CyberPredictor: Система прогнозирования киберугроз

**CyberPredictor** — это аналитическая платформа класса «система раннего предупреждения», разработанная для предотвращения киберинцидентов путем прогнозирования вероятности атак. В отличие от классических систем расследования атак, данный инструмент смещает фокус на упреждение угроз в ИТ-инфраструктуре заказчика.

## Ключевые фичи:
* **Многофакторное прогнозирование**: использование 48 моделей CatBoost, обученных для 4 типов инфраструктурных кластеров и 6 типов угроз;
* **Двойной горизонт планирования**: одновременный расчет прогнозов на 24 часа и 7 дней для оперативного и тактического реагирования;
* **Интерпретируемый ИИ**: визуализация влияния признаков на результат прогноза с помощью графиков SHAP для прозрачности принятия решений;
* **Автоматизированная отчетность**: генерация подробных md-отчетов с рекомендациями по минимизации рисков.

## Тех. стек:
Проект реализован в виде микросервисной архитектуры:
* **Frontend**: React, Vite;
* **Backend**: Node.js, Express, Prisma ORM;
* **Database**: PostgreSQL;
* **ML Service**: FastAPI (Python), CatBoost;
* **Infrastructure**: Docker, Docker Compose.

**Роль: Backend developer**

В рамках хакатона МФТИ в проекте «Система прогнозирования киберугроз» я отвечала за архитектурную связку сервисов и превращение ML-моделей в готовый продукт.

**Мой вклад:**
* **Проектирование API и интеграция**: реализовала взаимодействие между Node.js и ML, настроила параллельную обработку запросов для получения прогнозов по двум временным горизонтам одновременно;
* **SHAP**: разработала модуль генерации графиков важности признаков SHAP внутри FastAPI сервиса, реализовала механизм передачи изображений в формате Base64 через API для их динамического отображения на фронтенде;
* **NLP-lite**: написала адаптер для нормализации входящих данных от пользователя и их маппинга на кластеры инфраструктуры, используемые ML-моделями. Таким образом я решила проблему несовместимости названий отраслей и внутренних идентификаторов модели;
* **Работа с данными и БД**: спроектировала схему базы данных в Prisma и PostgreSQL, настроила миграции для хранения истории прогнозов и рекомендаций;
* **DevOps и контейнеризация**: настроила Docker-инфраструктуру проекта, написала конфигурацию Docker Compose для оркестрации базы данных, бэкенда и ML.
